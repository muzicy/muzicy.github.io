socket.io实现即时聊天窗口：

用socket.io node.js实现一个网页版的聊天窗口。其中碰到的最大问题就是在很多情况下需要调用function.bind(socket)来实现功能。因为每个页面的socket不一样，如果不限定在此socket.io下绑定事件，那么就很容易出错，比如在开发时总是容易碰到，非最后定义的窗口发布消息时自己能收到自己的消息，而最后定义的那个窗口无法接受到消息，找了好久发现原因在于，最后定义窗口时，服务器端socket保留的是最后那个页面的socket，因此在前几个页面发布消息时，调用socket.broadcast.emit()方法时，默认的socket保留的是最后一个页面的socket，因此出现只有最后一个窗口无法收到其它人发送的消息，而其它页面都能接收到自己发送的消息，就是因为socket默认绑定的是最后一个页面的socket.因此必须手动绑定socket实现函数， 通过bind来实现即可，即通过这样的方式：
socket.on(‘text’, getMsg.bind(socket));
function getMsg() {
this.broadcast.emit(‘text’,args);
return this;
}

v-if和v-show的差别

2017-04-27

v-if 和 v-show都是条件指令， 都可以通过true和false来切换显示的状态， 具体v-if和v-show的差别在哪里：

首先v-if是一种懒加载， 如果在检测到指令值是false时， 就直接什么也不会做， 只有在指令值是true时，才会加载和编译组件内的子组件和注册的事件。

v-show初始状态是不管状态是什么都是会首先渲染结构再通过display：none/block来显示组件， 因此具有较高的初始渲染消耗,之后也只是通过简单的display来控制显示。

v-if相对于v-show来说才是真正的条件编译， 只有在条件为true时才会工作， 因此相对于v-show来说具有更高的切换消耗， 而v-show具有较高的初始渲染消耗。

因此在涉及到频繁切换组件时，应该使用v-show指令，而运行条件不太可能更改的时候用v-if比较合适。

前端模块化

2017-04-25

说说之前和现在知道的模块化

  1. node.js 的模块化是Common.js规范， 就是每个文件就是一个模块，调用模块的时候就是直接let fs = require(‘fs’); 这种加载就是运行时加载。 模块输出有两种方式： 1 module.exports = moduleName 或者 exports.moduleFunc = function() {} 这两种方式实现,加载时调用require(‘./module.js’)加载。

2.前端模块化的方式es6之前的模块化：

1. 调用即时调用函数实现代码的封装， 可以避免全局污染，这也算是初步的模块的实现

2. AMD规范的模块化，通过require.js配置和加载模块。 模块里面用如下格式实现define('socket','b', function(socket,b) {    })， 在页面里面加载js文件通过data-main属性加载文件，如果有不支持AMD规范加载的模块，需要配置require-config.js里面的shim属性，
还有一个是遵守CMD规范的模块化，作为代表的就是SeaJs， AMD与CMD的不同之处在于，首先.AMD是提前执行，在模块开头就加载编译所需的模块， CMD是延迟执行，推崇使用时再加载编译.从AMD规范里面一眼就可以看出依赖了那些模块，CMD需要深入代码才能看到。 CMD的书写格式如下：
```bash

   define(function() {
          var a = require('./a.js')
          a.sayName();

          var b = require('./b.js')
          b.sayName();
       })
```
  1. 现在es6解决了前端模块化的问题， 一个模块就是一个文件，模块里面定义的变量外界无法获取，es6通过export 和 import 输出和导入模块中的变量。

I. export和import命令

eg. export let a = 10; export b =20 等价于export {a, b} ； 那么在另外一个文件里面就可以通过import {a, b} from ‘./test.js’来获取这两个变量。

export导出函数： export function add () {}; import {add} from ‘./test.js’ 注意的是import引入的变量名要与export输出的变量名相同，如果想要修改名字， 用as指令 ，如import {A as add} from ‘./test.js’，import还可以加载对象的部分输出接口 比如：export function a() {}; export function b() {}; import {a, b} from ‘./test.js’

II. module命令可以取代import语句，达到整体输入模块的作用， 用法： module test form ‘./test’

III. export default 为模块指定默认输出， export default function() {}; import加载是可以为该模块指定任意名字，现在vue-cli开发里面都是用的这种组件导出方式。
